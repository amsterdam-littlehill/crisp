# Workflow: Fix Bug

## 前置检查（必须先读）

1. `rules/project-rules.md`
2. `rules/coding-standards.md`
3. `references/gotchas.md`（扫描是否有相关坑点）

## 执行步骤

### 1. 定位问题

- 读取相关代码，理解当前行为
- 复现 bug，确认触发条件
- 标记需要修改的确切文件和行

### 2. 精准修改

**铁律**：只改必须改的，只清理自己造成的乱。

- 每一行改动都必须能追溯到用户请求
- 不加 docstring、不加强校验、不改无关代码
- 如果发现有连锁影响，先报告，不自行扩域

### 3. 验证

- 运行最小复现测试，确认 bug 已修复
- 运行现有测试，确认没有回归
- <!-- FILL: 项目特定的验证命令 -->

### 4. 30 秒 AAR 扫描

任务不算完成，直到回答这 4 个问题：

1. **新模式？** — 用了未记录的模式或约定吗？
2. **新陷阱？** — 遇到了不提前知道就会浪费大量时间的问题吗？
3. **缺失规则？** — 因为缺少某条规则导致走了弯路吗？
4. **过时规则？** — 发现现有规则已经不准确或不再适用吗？

**跳过条件**（仅以下情况可跳过）：纯格式化、纯注释、仅依赖版本变更、无新教训的重构。

如果任一答案为"是" → 走 `workflows/update-rules.md` 录入流程。
